공학컴퓨터프로그래밍 팀 프로젝트 기획서
“영상분류기”

1. 프로젝트 핵심 전략: "똑똑한 영상 파일 관리자"
AI가 내용을 분석하게 하면 엄청난 부하가 생길 것이므로, 파일 자체의 정보(메타데이터)를 100% 활용하고, 사용자가 수동 관리를 최대한 편하게 할 수 있도록 UX(사용자 경험)를 설계하는 것이 핵심이다.
•	방식: 사용자가 파일을 올릴 때 파일명, 날짜, 용량 등을 자동 추출하고, "중복 파일 찾기", "파일 확장자로 검색" 등의 편의 기능을 제공한다.

2. 기술 스택
•	GUI (화면): PyQt5 (Tkinter보다 디자인이 깔끔하고 기능이 강력)
•	데이터베이스: SQLite3 (파이썬 내장, 별도 설치 필요X) 
•	영상 처리/썸네일: OpenCV (cv2) (무료 라이브러리. 영상의 한 장면을 캡처하여 이미지로 저장하는 데 사용)
•	영상 재생: PyQt Multimedia 위젯 (외부 플레이어 없이 창 내부에서 재생 가능) 

3. 상세 기능 기획
A. [핵심?] 자동 썸네일 추출기 (OpenCV 활용)
•	기능: 영상을 DB에 등록할 때, 기존 영상에 이미 썸네일이 있다면 해당 썸네일을 사용하고, 그렇지 않을 경우 영상의 10% 지점의 프레임 하나를 캡처해서 이미지 파일로 저장하고 리스트에 보여준다(썸네일 자동 생성). 사용자가 원할 경우 썸네일을 수동으로 변경할 수도 있도록 한다.
•	기술 포인트: cv2.VideoCapture 함수를 사용하여 영상 프레임을 읽어오는 로직 구현. 이때 저장공간이 많이 요구되기 때문에 저화질로 압축하는 과정을 거친다.
•	장점: 파일명만 있는 것보다 시각적으로 훨씬 완성도 높은 프로그램이 된다.
B. [핵심] 메타데이터 자동 수집 (os, time 모듈 활용)
•	기능: 사용자가 태그를 입력하기 전, 이미 컴퓨터가 알고 있는 정보를 싹 긁어와 DB에 저장합니다.
o	파일의 최종 수정일
o	파일 크기 (용량 큰 순서대로 정렬)
o	재생 길이 (이건 OpenCV나 MoviePy로 추출 가능)
o	확장자 이전의 파일 이름
o	파일 원본의 위치
•	장점: 사용자가 일일이 입력해야 하는 번거로움을 줄여준다.
C. [차별화] 중복 파일 감지기 (Hash 알고리즘) 
•	기능: 사용자가 실수로 똑같은 영상을 다른 폴더에 저장했거나, 파일명만 바꿔서 가지고 있는 경우를 찾아낸다.
•	기술 포인트: 파일의 내용을 바탕으로 고유한 문자열(Hash)을 만드는 MD5 또는 SHA-256 알고리즘을 사용한다.
o	"파일명은 다르지만 내용은 100% 같은 파일입니다. 삭제하시겠습니까?" 라는 팝업 띄우기
•	장점: AI 없이도 프로그램이 '스마트하다'는 인상을 심어준다.
D. [편의성] 해시태그 시스템 및 검색
•	기능: 인스타그램처럼 #여행 #가족 형태로 태그를 저장하고, 검색창에 태그를 입력하면 해당 영상만 필터링 (SQL WHERE 문 활용)
•	구현 팁: DB 테이블을 설계할 때 Video 테이블과 Tag 테이블을 분리하여 1:N 관계 맺기. 
E. [편의성] 검색 시스템
•	기능: 태그뿐만 아니라 파일 이름, 최종 수정일(날짜), 파일 확장자 유형(mp4, avi, mkv 등), 영상 길이에 따라서 필터링하여 검색이 가능하게 한다.
•	구현 팁: SQL WHERE 절 활용. 데이터베이스에 저장된 메타데이터(파일명, 날짜, 확장자, 길이)를 기준으로 LIKE 연산자, 비교 연산자 등을 사용하여 조건에 맞는 영상을 조회하는 로직 구현.
•	장점: 사용자가 원하는 영상을 다양한 조건으로 신속하게 찾을 수 있도록 편의성을 극대화한다.
F. [편의성] 리스트에 나와 있는 파일 이름 우클릭 기능
•	기능: 리스트에서 파일 이름을 우클릭 했을 때, 파일 이름을 수정하는 기능과 해당 영상 파일이 위치하고 있는 경로를 여는 기능 등을 제공한다.
•	구현 팁: 
1)	경로 열기: os 또는 subprocess 모듈을 사용하여 DB에 저장된 파일 원본의 위치를 OS의 파일 탐색기/Finder로 열어주는 명령어를 실행한다.
2)	이름 수정: os.rename() 함수를 사용하여 디스크의 실제 파일 이름을 변경하고, 동시에 SQLite DB의 해당 레코드를 UPDATE 쿼리로 갱신한다.
•	장점: 사용자가 원하는 영상을 다양한 조건으로 신속하게 찾을 수 있도록 편의성을 극대화한다.

4. 개발 로드맵
기본 골격 및 DB 설계
•	SQLite DB 파일 생성 및 테이블 설계 (Video 정보 저장용)
o	Video 테이블: 파일명, 날짜(최종 수정일), 크기, 재생 길이, 확장자, 파일 원본 위치 등을 저장
o	Tag 테이블: 해시태그 저장 및 Video 테이블과 1:N 관계 설계
•	os.walk를 이용해 특정 폴더 내의 모든 .mp4 파일을 찾아 리스트로 뽑는 파이썬 스크립트 작성
•	메타데이터 자동 수집 로직 (B. 항목) 구현: 파일명, 날짜, 크기 등 기본 메타데이터 추출 및 DB 저장
UI 및 영상 처리 연동
•	PyQt5로 기획서(PDF 4~6페이지)에 있는 레이아웃 만들기
•	OpenCV를 연동해 리스트에 있는 영상들의 썸네일을 따서 UI에 이미지로 띄우기
•	리스트 클릭 시 영상 재생 (내장 플레이어 또는 새 창 띄우기)
검색, 재생 및 디테일
•	검색창에 글자 입력 시 DB에서 필터링해서 보여주는 기능
•	우클릭 메뉴 구현
•	(여유되면) 중복 파일 감지 기능 추가
•	UX 디테일 마무리
